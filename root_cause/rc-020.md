## Description
テスト容易性を意識した設計やアーキテクチャ戦略が初期段階から不在である。

## Context
設計時にテスト観点での分離や検証容易性を考慮しておらず、責務の分割・データ依存排除・モック可能な構造などが欠落しているケースが多い。  
また、テストのしやすさに関する非機能要件が明示されず、チーム全体での共通理解も不足している。

## Impact
開発後期でのテスト計画が破綻し、バグ検出・修正・再検証のサイクルが長期化し、納期遅延や品質低下につながる。
