## Description
ビジョンが定義されておらず、システムやアーキテクチャの設計方針に一貫性が欠けている。

## Context
経営層やビジネス部門との間で中長期的な方向性が合意されておらず、プロジェクト開始時点での設計判断がブレやすくなる。  
また、要件定義やレビューの際に「何が正解か」が判断できない状態に陥りやすい。

## Impact
実装された成果物がビジネスニーズに適合せず、度重なる修正や認識齟齬によるプロジェクト遅延・コスト超過を招く。
