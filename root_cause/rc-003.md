## Description
DevOps、インフラ、アプリ運用の責任範囲が不明確なため、設計・構成・運用の連携が破綻しやすくなる。

## Context
例えば「誰が何をモニタリングすべきか」「どこまでがコードとして管理されるか」といった認識が曖昧で、各チームがサイロ化して作業している場合が多い。

## Impact
連携の不備により障害対応や改善活動において属人化・手戻りが発生しやすく、**設計方針の不整合や運用上の抜け漏れ**が多発する。
